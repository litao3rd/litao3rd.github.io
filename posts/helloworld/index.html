HelloWorld

2021-10-16
3 min read

This is another HelloWorld.

What’s this

HelloWorld 可能是我写的最多的东西了，学习新的编程语言的时候我会写一个HelloWorld，学习新的框架的时候也是HelloWorld，断断续续构建过几次博客的第一篇文章也是HelloWorld，可惜的是之前的博客都没有坚持下来，这次又是一篇HelloWorld。

这是一个使用 markdown 标记语言编写，使用 Hugo 生成静态html搭建的博客系统，使用的主题是 hugo-eureka，这个主题很精简，也能实现相当多的功能，比如支持按照文档格式组织文章结构，也支持简单的版本控制，基本满足我的需求。

博客现在部署在 GitHub Pages 系统上，我在 github 创建了两个相关的仓库，第一个仓库是一个私有仓库，所有的 markdown 代码以及博客系统主题等其他配置信息都会保存在这个仓库中。第二个仓库是一个公开的仓库，这里保存的是 hugo 生成的静态网站代码文件。在私有仓库中，我配置了一个简单的 Github Actions，当私有仓库的 master 分支有提交事件时，将会触发执行 workflow，生成结果文件并且提交到公有仓库以提供访问。

在本地我使用 Typora 作为 markdown 编辑工具，我非常喜欢 Typora 的所见即所得的功能，性能也很不错，跨平台支持也很棒，基本能覆盖我的所有本地设备。

当前我还没有插图的需求，更多的是文字和代码片段，等后面对插图有强烈需求时，我可能会选择一个比较合适的图床或者直接将图片提交到 Github 仓库。值得一说的，Typora 和 Hugo 都支持 mermaid.js，这也是一个很棒的工具，有点类似于 Graphviz，支持通过代码生成流程图，这对于缺少审美和作图能力的我来说是一个很棒的工具，我很喜欢。

Why another

年纪大了，记性不太好了，以前从来不相信什么好记性不如烂笔头，对自己的记性有着无与伦比的自信，可是现在发现很多东西需要靠搜索引擎、笔记来提示自己才能想起来，而伴随着各种信息孤岛的产生，很多时候搜索引擎带来太多的噪音了，我又很讨厌一遍一遍的做重复的事情，发现可能烂笔头还是有点用的。

另一方面，记录下来也是有助于将来回顾的时候更好的感悟自己的成长。工作几年，发现以前对很多东西的理解都很粗陋，随着使用经验以及年岁的增长，很多原来并不明白的东西也有了新的感悟，现在看自己三个月之前的代码都会觉得当时的思维很幼稚，即使在当时看来那个设计是非常满意的。

把自己的想法记录下来，这能督促自己学习总结，同时也是很好的一种考验自己自律能力的一种方式，虽然我已经失败了很多次，但是我还是要站起来再试一次。
